# Doppio Architecture Overview

## What is Doppio

Doppio is a Progressive Web App (PWA) that takes non-technical workers — office professionals, managers, small business owners who currently use ChatGPT like a search engine — and transforms them into confident AI coworker managers in 20 minutes, using only natural language. It does not teach "prompt engineering"; it curates and sequences existing high-quality TikTok/YouTube Shorts social video demos into a structured 3-level learning path (9 video cards total), where each card pairs an embedded social video with a "Try it" CTA that opens the relevant AI tool (ChatGPT, Claude, or Perplexity) in a new tab with a single prefilled natural-language prompt. No backend AI processing, no user accounts or login, no app store — a shareable URL with a fully offline-resilient PWA is the entire delivery mechanism.

## Tech Stack

| Layer | Technology | Notes |
|-------|-----------|-------|
| Framework | React 18 + Vite | SPA, no SSR |
| Styling | Tailwind CSS | Utility-first; progress bar is CSS-only via Tailwind |
| PWA | vite-plugin-pwa v0.21.x + Workbox | CacheFirst for app shell, autoUpdate SW |
| PWA icons | @vite-pwa/assets-generator | Single source PNG → all icon sizes |
| Database + Auth | Supabase (Postgres + Auth) | Anonymous auth only; no login required |
| Analytics (layer 1) | Vercel Analytics (@vercel/analytics) | Automatic page views, referrers, geo, device |
| Analytics (layer 2) | Supabase custom events table | 7 tracked events (see Key Data Flows) |
| Routing | React Router v6 | Client-side only |
| Video (YouTube) | lite-youtube-embed web component | Facade/lazy-load pattern |
| Video (TikTok) | Direct iframe URL | `https://www.tiktok.com/embed/v2/{VIDEO_ID}` |
| Confetti | canvas-confetti (~6 kB gzipped) | Level completion only |
| Deployment | Vercel Hobby (free) | Custom domain: `doppio.kookyos.com` |

## Key Data Flows

### 1. Progress State: localStorage -> Supabase

```
User completes a card
        |
        v
Write to localStorage immediately (source of truth, non-blocking)
        |
        v
UI updates instantly (progress bar advances, card shows completed state)
        |
        v
Background: supabase.from('user_progress').upsert({ user_id, level, card })
        |
        v
On window.focus: pull from Supabase → merge with localStorage (union/additive)
        |
        v
If Supabase unavailable: silently skip sync; localStorage remains source of truth
```

Supabase schema:
```sql
create table public.user_progress (
  id           uuid default gen_random_uuid() primary key,
  user_id      uuid not null references auth.users(id) on delete cascade,
  level        smallint not null check (level between 1 and 3),
  card         smallint not null check (card between 1 and 3),
  completed_at timestamptz not null default now(),
  constraint unique_user_level_card unique (user_id, level, card)
);
```

### 2. Analytics Events: Component -> Supabase table

Seven tracked events fire via `logEvent()` in `src/lib/analytics.ts`:

| Event | Fired when |
|-------|-----------|
| `page_view` | Any route change |
| `level_started` | User enters a level screen |
| `card_completed` | User taps "Mark as done" on a card |
| `try_it_clicked` | User taps "Try it" CTA on any card |
| `level_completed` | All 3 cards in a level are complete |
| `badge_shared` | User taps Share on completion screen |
| `pwa_installed` | PWA install prompt accepted |

Supabase schema:
```sql
create table public.analytics_events (
  id          uuid default gen_random_uuid() primary key,
  event_name  text not null,
  session_id  uuid not null,
  properties  jsonb,
  created_at  timestamptz not null default now()
);
```

Vercel Analytics fires automatically on every page view via the `@vercel/analytics` package (no custom code needed for layer 1).

### 3. Video Cards: content.json -> Components

```
src/data/content.json
        |
        v
Imported statically by LevelScreen.tsx
        |
        v
Passed as props to VideoCard.tsx
        |
        v
VideoCard renders facade (static thumbnail + play button)
        |
        v
On user click: real iframe loads (lite-youtube-embed or TikTok iframe URL)
```

The `content.json` structure drives every card. Video IDs can be swapped without touching any React component or triggering a code redeploy — only the JSON file changes.

## Key Design Decisions

### localStorage-first for progress (offline resilient)
Progress is written to localStorage synchronously on every card completion. The UI never waits on a network call. Supabase sync is fire-and-forget in the background. If the user is offline, everything still works. Merge strategy on reconnect is additive (union): cards are never un-completed.

### content.json for all video/prompt content (swappable without redeploy)
All 9 video IDs, card titles, level titles, "Try it" prompts, and target AI tool URLs live in `src/data/content.json`. This was a hard decision from DISCOVERY.md (D20). If a video is removed from YouTube or TikTok, the operator updates one JSON file; no component code changes and no redeploy of JS bundles is required (Vercel can serve the updated JSON from the CDN edge on next request).

### Facade pattern for video embeds (performance)
YouTube and TikTok iframes are 10 MB+ of third-party resources on initial load. The facade pattern shows a static thumbnail with a play button overlay. The real iframe is inserted into the DOM only when the user clicks. This eliminates CLS (cumulative layout shift) and dramatically reduces initial load weight. YouTube uses `lite-youtube-embed`; TikTok uses a direct iframe URL mounted on click.

### Anonymous Supabase auth (no login required)
`supabase.auth.signInAnonymously()` is called on first app load. This gives each device a stable `user_id` UUID that ties progress rows in `user_progress` to the device without any email/password or OAuth flow. If this call fails, the app silently falls back to localStorage-only mode — the user never sees an error.

### CSS-only progress bar (no library)
The progress bar is a standard HTML div with Tailwind utility classes (`transition-all duration-500 ease-out`) and an inline `width` style driven by `(completedCards / totalCards) * 100`. No charting library, no animation library dependency. Keeps the bundle small and avoids a third-party component that would need styling overrides.

### canvas-confetti for level completion only
Confetti fires exactly 3 times maximum — once per level completion screen. It never fires on individual card completions. This preserves the delight of the "finale" moments and avoids desensitizing the user. Mobile-safe parameters: `particleCount: 80-120`.

## Routing

React Router v6 client-side routes:

| Route | Component | Purpose |
|-------|-----------|---------|
| `/` | LandingPage | Hero + teaser video + "Start Level 1" CTA |
| `/learn` | LevelScreen | Active learning path (current level's 3 cards) |
| `/complete` | FinalComplete | "You're an AI Manager!" + share badge + resource links |
| `/?ref=badge` | LandingPage | Same landing page with a badge referral banner shown |

There is no traditional navbar. The top bar shows the Doppio logo and a global progress indicator only. Back navigation uses the browser's native back button.

## Environment Variables

Both variables are public and safe to expose client-side. The Supabase anon key is designed for browser use — it is not a secret. All variables are prefixed with `VITE_` so Vite exposes them to the client bundle.

| Variable | Value pattern | Required |
|----------|--------------|---------|
| `VITE_SUPABASE_URL` | `https://<project-ref>.supabase.co` | Yes |
| `VITE_SUPABASE_ANON_KEY` | `<anon-key from Supabase Dashboard>` | Yes |

Set in `.env.local` for local development. Set in Vercel project settings (Environment Variables) for production. Never committed to version control.

## Key Files Every Agent Should Know

| File | Purpose |
|------|---------|
| `src/data/content.json` | Single source of truth for all video IDs, card titles, prompts, and AI tool URLs. Change content here, not in components. |
| `src/lib/supabase.ts` | Supabase client singleton. Handles `createClient()` init and `signInAnonymously()` call. Import this to get the `supabase` client instance. |
| `src/lib/progress.ts` | All progress read/write logic. Encapsulates localStorage key structure, Supabase upsert, and merge/union strategy on sync. Use `useProgress` hook for component access. |
| `src/lib/analytics.ts` | `logEvent(name, properties)` function. Writes to Supabase `analytics_events` table. Vercel Analytics is separate and automatic — this file covers custom event tracking only. |
